Folks,

My previous mail asked a direct question about filtering a socket to
receive just error frames, but I should perhaps have asked a slightly
different question.

What I would really like to do is be able to tell after each frame
written to a socket whether it has been transmitted successfully.
Obviously what prompted my previous question was the idea that I could
just check for errors, but this raises the question of how long one must
wait for an error on the error socket before a transmission is deemed to
have 'succeeded'.  

So my question should have been 'is there a way using socketcan to tell
if a frame has been transmitted successfully' ?  

As a prototype I've added the error class CAN_ERR_NONE to
linux/can/error.h, and have the vcan driver return this in an error
frame if the transmission succeeds - but I'd like to know if there is a
real way to achieve the same thing.
