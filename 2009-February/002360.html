Hello,

here is a brief status of our effort to include the Socket-CAN device
interface and drivers into the kernel. The first patch series I posted
last week to the netdev mailing list has been reviewed and discussed.
Unfortunately, we need to check and change various things:

- Abandaon the SysFS in favor of a netlink interface as described here:

    http://lwn.net/Articles/236919/

  I'm not sure about the CAN statistics, though. Netdev people would
  like to see CAN support in the "ip" utility of iproute2. Then the CAN
  device would be started as shown:

    ip link set can0 bitrate 125000 up

- Use container_of() for localizing the outer private data structure for
  the device.

- Check and fix locking issues.

Still plenty of work to be done. Next week I will be on the
EmbeddedWorld fair in Nuremberg. I hope to find some free time thereafter.

Wolfgang.
