Hi Wolfgang,

after reading a bit in the CC770 spec i wonder, if having the CC770 additional
functions to be 'on' by default is the correct approach - especially because
the CTRL_EAF bit should not be set to '1' when having the AN82527.

I assume most hardware out there uses the AN82527 these days ...

I would suggest to add the configuration bits for EAF and the msgobj15_eff
somewhere into struct cc770_priv and add 'eaf' or 'cc770_eaf' and
'msgobj15_eff' as module parameter array(!) to cc770_isa.

Although 'cc770_eaf' should be off (== 0) by default.

What do you think about this?
