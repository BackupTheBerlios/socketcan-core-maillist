Hi,

here is an (almost) trivial patch to add support for the Socket-CAN API
to CanFestival-3 (CVS). I also fixed and cleaned up the Xenomai timer
layer. A bit unclear to me is the handling of extended frames, so I
followed the lincan pattern (IF id >= 0x800 THEN extended).

The basic TestMasterSlave works fine here over Xenomai (except for
occasional timeouts - but that's due to my qemu environment :)). I
didn't test the plain Linux variant, but I don't expect troubles as well.

While hacking my local Xenomai path into configure and extending the
CONFIG switches, I wondered if there is a particular reason for the
hand-coded build system of CanFestival. Any supported platform where
autotools don't fly?

Jan
