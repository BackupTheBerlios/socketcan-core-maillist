> Hm - i like excessive parentheses :-)

When you were in school, did you learn that

   (a+b)^2 = a^2 + 2ab + b^2    or     ((b^2) + (2(ab)) + (b^2))

and which one do you find easier to read?  Operator precedence and
rules for parentheses elimination are there for a reason.

The more parentheses you use, the more you have to read, parse, and
find matching parentheses, which makes code harder to read.  In your
code I've often seen expressions that begin with 4 or 5 ( and end with
the same number of ), where one or two would suffice.  I you make a
mistake in such code it is usually much harder to find wrong
parentheses than missing parentheses.
