I would suggest to hide all the possible stuff like (fixed) tables and 
the (simple) bitrate calculation behind /sys/.../can_bitrate like this:

Whenever the user sets the bitrate via /sys/.../can_bitrate, the driver

1. checks whether he has a matching built-in value (in a table)
2. if not available: make a simple bitrate calculation
3. if not calculable: return an appropriate error value to the 
sysfs-write call

This looks quite straight to me and does not overextend the user and the 
user interface ...
