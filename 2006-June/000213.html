Andrey Volkov <avolkov at varma-el.com> writes:

>  Fix "warning: suggest parentheses around assignment used as truth value"

It makes the code ugly and less readable.  Using assignment in if,
for, while, etc. is *very* common C idiom and needs no extra
parentheses.  GCC's -Wparentheses option is a tribute to C beginners
which often come from Pascal or similar background where = is
comparison and := is assignment.  Any proficient C programmer knows
without thinking what the meaning of these C operators is.
