On Mon, Jun 23, 2008 at 03:38:38PM +0200, Kurt Van Dijck wrote:
> This patch adds the softing Gmbh CANcard to the socketCAN list.

Cool!

> Things you should know on this driver:
> * Checkpatch.pl has a few warnings on complex macro's

Why do you need complex macros?

> * I tested with kernel 2.6.23.8

Please re-test on linux-latest. We try to make things ready for
mainline, so anything that works at least on 2.6.26-rc is a good thing.

> * It won't work on a big endian machine (yet)

Looks like hardware access has to be reviewed.

> * The drivers looses knowledge of which socket initiated which packet,
> 	so CAN_RAW_RECV_OWN_MSGS won't work
> * status: worked for me,

uuuuuh...

> In order to use the pcmcia card, you need firmware.
> I didn't hear anything from Softing Gmbh yet, but noticed that the
> firmware in the form of byte-array is released under gpl.

You can't release byte array firmware under the GPL. The GPL requires
that the code is in a format which is the "primary" format for editing.
But the kernel has a standard interface for binary firmware and this
should be used.
