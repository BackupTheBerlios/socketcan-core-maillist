> Hartkopp, Oliver (K-GEFE/E) wrote:
>> Hi Wolfgang,
>>
>> Wolfgang Grandegger wrote:
>>> Hello,
>>>
>>> what is the default behavior of a socket if no filter has been defined 
>>> with setsockopt? Should it receive all messages by default or none?
>> It receives all messages. If you make any setsockopt()-syscall to set
>> filters BEFORE the bind()-syscall, you will only get the wanted messages.
If
>> you just say bind() you will get everything.
> 
> OK, and if I just want to send messages to the socket? Is there a 
> write-only mode?

:-) If you do not want read anything, it's a good idea to set the
appropriate filter to receive nothing. 

e.g.

struct can_filter myfilter;

myfilter.can_id = CAN_INV_FILTER;
myfilter.can_mask = 0;

setsockopt(mysocket, SOL_CAN_RAW, CAN_RAW_FILTER, &myfilter,
sizeof(myfilter));
 
can_mask = 0 leads to all frames to pass the filter BUT due to
CAN_INV_FILTER you will get nothing.
