Hans-Jürgen Koch wrote:
> Am Fri, 18 Jan 2008 12:12:58 +0100
> schrieb Wolfgang Grandegger <wg at grandegger.com>:
>
>   
>> Hello,
>>
>> while progressing with the CAN device interface, I asked myself if we
>> really need a per device parameter to configure echo-skbs and
>> restart-ms. I think global parameters are enough, e.g. a module
>> parameter in dev.c which could be modified via:
>>
>> /sys/module/can_dev/parameters/echo
>> /sys/module/can_dev/parameters/restart_ms
>>
>> What is your opinion.
>>     
>
> It should definitely be a device option. It is too obvious that somebody
> can have two busses with different settings of these parameters. I
> remember we had a long discussion about the usefullness of auto restart
> and found sensible applications for both with and without restart.
>   

Yes. I would also like to follow Hans and Marc. These settings should be
per device. Additionally there might be some devices that do not support
(or have a fixed setting) for one of these parameters. E.g. the slcan
driver will never support proper echo handling for sent frames. So if
you run with mixed CAN hardware (slcan, peak_pci, homebrew i82527)
you'll run into multiple problems with a 'global' setting in can_dev.
