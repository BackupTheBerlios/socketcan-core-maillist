Pavel Pisa <pisa at cmp.felk.cvut.cz> wrote:

> In the fact, I have done it such way, that new sja1000_mem can handle both
> IO and MEM. It is questionable if new kernel option to use single I/O
> access macros for both spaces should be used there instead.

You should keep in mind, that at least on x86, there are devices which use
use a third way of SJA1000 Register Access. It is usually called indirect
IO. It uses two IO-mapped registers, address and data register.

First you write the SJA1000 address to the address register then you are
able to access data via data register.

I used to solve the Problem of conditional jumps by using function pointers
for register access instead of direct IO function calls. The function
pointers. This way, the function pointers can be initialised whith the
correct IO-method on module load time.
