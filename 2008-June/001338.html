> Setting controller specific things like SJA1000' synchronization jump
> width, is practically not done even not by the so-called CAN experts.
> Almost all customers of us uses our standard bitrates for CiA recommended
> bitrates.
> 

I agree on that.

> If someone, want's to change really controller specific things, he may
> change this setting via a module parameter?!
> 

This is IMHO the only way it will work. But we have to make sure that
this module parameter is evaluated by the specific hardware driver for
the CAN interface, not inside the controller driver. E.g. the ems_pci.ko
module is responsible for the bit timing setup, not the sja1000.ko
module.

Concerning the "simple" bitrate calculation: there is no "generic"
implementation possible. Just use the following link to get an
impression about a few number of controllers:
http://www.port.de/engl/canprod/sv_req_form.html
