i just assumed that you also subscribed the socketcan-commit mailing 
list ... sorry.

I already implemented your suggestion at November 17th. It's stable 
since November 19th ;-)

It was a good suggestion from you as it also lead to have a more 
straight & clean implementation for the filters now. Now the default 
filter is 'get anything' but you are able to set a filter of length zero 
now. If you change the filter-setting away from the default filter, 
there is currently no sockopt to restore the default filter. You have to 
set the 'receive all' filter by hand, if(!) needed. I've been discussing 
with Urs, if we would need an additional sockopt to 'restore' the 
default filter. What do you think on this topic? In my opinion it's the 
users responsibility to set whatever he want's ... i personally would 
not need any 'restore default filter' sockopt for my work.

Please compare Revision 101 to 99 to find the relevant changes. I also 
modified a test-application for CAN_RAW-sockets, so that you should be 
able to 'play' with the settings from the commandline.

Best regards & thanks for the suggestion,

Oliver
