> Hallo Wolfgang,
> 
> in can_calc_bittiming() i moved the lines with "real sample point" some line
> up in the code to match the current mainline source code.
> 
> Accidentally i forgot to add the conditional compilation depending on
> CONFIG_CAN_DEV_SYSFS ... so i reverted this code movement again - sorry ...
> 
> Can you please take a look if the assignmens of
> 
> bt->sample_point
> bt->bitrate
> 
> are really correct in their position inside the code (which is different in
> mainline) and if it's really needed to have these two assignments in
> 
> "#ifndef CONFIG_CAN_DEV_SYSFS"
> 
> areas?

Yes, there was a good reason to do so ;-). With SysFS we have the
problem, that each value must be set individually. If we set
bt->sample_point to the real value, it will affect a resetting of the
bit-timing. Otherwise the SysFS interface would required setting the
values in a defined order. Like it is, it behaves like before.
