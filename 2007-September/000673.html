Chris Elston wrote:
> What I would really like to do is be able to tell after each frame
> written to a socket whether it has been transmitted successfully.
> (..) So my question should have been 'is there a way using socketcan to tell
> if a frame has been transmitted successfully' ?  
>   

Hi Chris,

please keep in mind, that you're working with a Linux networking
solution, that normally even does not inform the user(s!) from bad
ethernet frames (until it doesn't lead to a timeout in a higher level).

Assume the things to be ok until you get an Error. Everything else is
(not even good) deep embedded style. What can you *really* do with this
kind of information?? You can react on Errors - nothing more.

Although (if you want to be really sure) you may use the loopback
functionality on the driver level (see Documentation/networking/can.txt)
that only loops back sent frames only on a successful TX-interrupt.
