> Auto recovery of the CAN controller is not allowed according ISO 11898-1.
We have to differ between non-standard conforming CAN controllers which
recovers without request like the MCP2510 or the Atmel and user
applications requesting bus-off recovery.

Doesn't this mean we have to stop the controller from going bus-on again
in order to get a standard conforming implementation?

> I'm not sure of if the workaround you have described above will work.

into reset mode before it is going bus-on again. And putting a
