Philipp Keller wrote:
> A question regarding rtcan:
> -is it possible to send to and receive on the same rtcan socket handler
> from different threads of a program, or receive in one thread and send
> in another? Is it thread safe?

It is. The first version of the rtcan utils made use of this by creating
the socket in the main thread and using it in a real-time thread - until
we folded both threads together for the sake of simplicity.
(You could even use it across processes due to the flat name space of
RTDM handles, but you shouldn't rely on this for portability reasons.)
