Hi all,

triggered by Daniel Schnell i thought about creating patches with all 
the new CAN drivers for current Kernels like 2.6.25, 2.6.26 and 2.6.27 
which originally already have CAN support.

I created a new file list 'FILES-2.6.25x-DRIVERS' that only contains the 
CAN driver stuff.

So if you would like to patch your 2.6.25 Kernel with the latest CAN 
drivers:

~/socketcan/trunk$ ./mkpatch 2.6.25 < FILES-2.6.25x-DRIVERS > 
/tmp/2.6.25-drivers.patch

or
~/socketcan/trunk$ ./mkpatch 2.6.26 < FILES-2.6.25x-DRIVERS > 
/tmp/2.6.26-drivers.patch
~/socketcan/trunk$ ./mkpatch 2.6.27 < FILES-2.6.25x-DRIVERS > 
/tmp/2.6.27-drivers.patch

The e.g. 2.6.27-drivers.patch can be applied in your vanilla Kernel e.g. 
with:

root at box:/usr/src/linux-2.6.27.4# patch -p1 < /tmp/2.6.27-drivers.patch

Than you can run 'make menuconfig' and select the CAN drivers you want.

Info for the experts: The version depended base patches in 
patches/2.6.2x.diff first remove the existing vcan files which are in 
the mainline Kernel before the big driver patch is applied.

Have fun,
Oliver
