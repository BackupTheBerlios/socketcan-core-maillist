Hi Oliver,

Oliver Hartkopp wrote:
> Just a short question for further discussions:
> 
> We have different source code and modules that are needed to have a 
> complete CAN netdev driver, e.g.:
> 
> - drivers/net/can/dev.c
> - drivers/net/can/sja1000/sja1000.c
> - drivers/net/can/sja1000/ems_pci.c
> 
> But how are these three parts named correctly?

That's an issue, indeed. I'm also not happy with the current names for 
the drivers. This also includes the name of the registered driver.

> - drivers/net/can/dev.c
> (can-dev.ko: CAN netdevice library)

With prefix "can-". That's fine.

> - drivers/net/can/sja1000/sja1000.c
> (sja1000.ko: CAN controller driver?)

OK, I think. A prefix "can-" would be an option but that seems not to be 
common (for network drivers).

> - drivers/net/can/sja1000/ems_pci.c
> (ems-pci.ko: CAN board driver?)

I think it should be sja1000-ems-pci. And I clearly prefer the file name 
being the *same* than the name of the module:

  sja1000-ems-pci.c
  sja1000-ems-pci.ko
  DRV_NAME "sja1000-ems-pci"

Currently we have:

  ems_pci.c
  ems_pci.ko
  DRV_NAME "can-ems-pci"

> Would 'controller driver' and 'board driver' be a correct naming for the 
> given examples within our discussions?

I think so.

> Or are there already defined names for that? E.g. something with platform*

What exactly are you thinking of.

Wolfgang.
