Jan Kiszka wrote:
> Wolfgang Grandegger wrote:
>> You have to bind the socket to an interface. Otherwise ifindex is 0
>> which could mean sending to all interfaces. But it's not implemented in
>> my RTCAN. Nevertheless, is this something we would like to support?
> 
> I'm not sure if send-to-all is that important (at least for the first
> run). But if you do not bind, sentto/sendmsg should still work fine and
> avoid overhead on reception side.

The binding is /mandatory/ to register the appropriate socket to the
can-core to receive CAN-Frames. Using ifindex=0 leads to receive from every
CAN-network-interface that's 'up'. This was a requirement we integrated in
the Socket-API at the beginning of this year.

To send CAN-frames you have to use e.g. sendto() with a valid ifindex in the
sockaddr_can if the socket is bound to ifindex=0. If the socket is bound to
a valid ifindex you may also use write().

Urs, i'm not that sure, but i think it should be possible to write
CAN-messages with sendto() even if the socket is not bound. Correct?
