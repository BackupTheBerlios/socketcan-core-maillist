Hello,

the current build system used in the can-utils has IMHO some drawbacks,
packaging isn't supportedand and external set CFLAGS are not obayed.

Speaking from a distribution maintainer point of view (propper)
autotoolized packages are far more easy to handle than hand crafted
makefiles.

While hacking on it I noticed an obsolete bzero in cangw, which is
replaced my memset in the first patch. I imported the svn to a
git repo[1], the second patch adds a .gitignore.

The third patch provides a autoconf/automake/libtool infrastructure.
It uses GNUmakefile instead of Makefile, so that it doesn't interfere
the existing Makefile.

Please review, test, comment and consider to apply.

cheers, Marc

P.S.: I'm willing to learn cmake if someone preferes it over autotools.
