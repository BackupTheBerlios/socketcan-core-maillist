Sorry to resurrect what you probably thought was a solved issue, but I
remain unconvinced by the argument that single-shot mode is a bus-level
setting.  I agree with Andrey that this is much more like a UDP vs. TCP,
and ethernet interfaces are not configured at the bus level to be either
UDP or TCP - that is done on a per-socket level for each bound socket.  

I think that configuring single-shot mode on a per-bus basis precludes
the case where you might want to mix both on the same device.  Setting
aside the use cases that we are aware of, what if someone came along
with a new use case that required the option of mixing the transfer
types on a bus?  For me, the per-socket setting is a more inclusive
strategy, in terms of what it allows the user to do.  It doesn't make
sense to restrict users unnecessarily when we are able to avoid that.
