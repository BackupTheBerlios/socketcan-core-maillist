Hi all,

the net_device_stats are handled in net/can/dev.c for all CAN devices 
(and various Kernel versions).
The current CCAN driver assumes the struct net_device_stats to be 
*always* in the priv data of the device (in struct can_priv) which is 
not true in Kernel 2.6.23+
This patch fixes the net_device_stats access, which is then analogue to 
the stats handling of the new sja1000 driver.

Should something like this also go into the old ccan driver in 
trunk/kernel/2.6/drivers/net/can/old/ccan/* ??

Regards,
Oliver
