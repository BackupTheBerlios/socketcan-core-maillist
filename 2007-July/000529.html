We have changed net/can/dev.c, which I will check in today.  So you
might want to wait with your patch and adapt it to the new dev.c.
We mainly changed the source to not look as we have two kind of
devices for each hardware interface, a net_device and a can_device,
and some functions working with one device and some functions with the
other device, and macros converting from one to the other.  Instead we
want to have only struct net_devices with a common private data
structure in dev->priv for all CAN net devices.  All functions should
only work with the usual struct net_device pointers.
