Let's sort the objective reasons:

- we need a method to specify a set of bit timing parameters manually
- one such set needs to be selectable
- a bit timing set has to be generated, either from a table or by an
  algorithm (running in userspace or kernel)
- *relying* on certain knowledge in userspace is *bad* because userspace
  tools bitrot and have to be kept in sync with the kernel. The more we
  are stand-alone, the better.

> > - there will be a built-in bitrate algorithm in dev.c to set the bitrate
> ... via "echo 125000 > /sys/class/net/can0/bit_rate

Still not preferred here. I want "125000" to be an handle for one
pre-defined set of bit-timings, how ever it was created (automatically
or manually).

I've also still not seen an argument *against* this.
