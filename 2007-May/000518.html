On Tue, May 22, 2007 at 03:06:50PM +0200, Urs Thuermann wrote:
> That produces one single patch whereas with quilt I build a patch
> series of smaller patches that are more likely to be accepted for
> mainline integration.  See
> http://www.spinics.net/lists/netdev/msg31663.html
> 
> I had started to think about a small awk (or perl) script that would
> generate a patch stack from a checked-out working copy without having
> a complete kernel source tree, but then found it would be too much
> effort compared with the few steps required to do this by hand using
> quilt.  The script would have to be taught what parts of the working
> copy go into which patch, and on a sub-file level since e.g.
> net/can/Kconfig is patched into the tree in several parts.

What about using an svn checked in quilt stack as the primary source,
now that the process of mainlining has started? We do it this way for
our kernel ports and it works pretty well. The pace is high and a clear
workflow is absolutely necessary when people start posting patches
against these patches.
