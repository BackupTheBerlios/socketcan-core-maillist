I agree. But I don't like the idea to have a table for different clocks
for the same controller.

Isn't the standard model an abstraction of the bitrate configuration for
each controller, and hence the controller bitrate configuration a concrete
implementation of this abstraction. So it must be possible to generate the
btregs out of the given standard model parameters for all clocks incl.
exotic ones?

We would just have one table of bitrates which consists of the standard
model parameters like sampling point, bit time. The controller must then
create it's own btregs out of it.
