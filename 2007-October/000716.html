I think that's incorrect. The five operation modes are Normal, Sleep,
Loopback, Listen-only and Configuration.  These are set using REQOP0-3
in CANCTRL.  One-shot mode has it's own configuration bit OSM in the
same register.  The manual does not state that you must enter config
mode to turn One-shot mode on.

The usage of one-shot mode is described in section 3.4, which is in the
chapter titled "message transmission".  The operation modes are
described in their own chapter , 10. "modes of operation".

I think it's a perfectly valid thing to do to from a hardware
perspective to turn one-shot mode on for a single frame, without having
to enter config mode first.
