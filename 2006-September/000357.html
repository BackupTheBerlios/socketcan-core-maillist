"Hartkopp, Oliver (K-GEFE/E)" <oliver.hartkopp at volkswagen.de> writes:

> Remark: The can.ko in r80 has to be loaded into the kernel before bringing
> CAN-interfaces up (due to creating data structures within the notifier). So
> there has to be an additional check for existing CAN-interfaces at
> init-time, which is already on the list ...

I've just read the src for register_netdevice_notifier() again.  We
don't need to do anything in the can_init() module init function.
register_netdevice_notifier() not only registers the notifier_block in
the call chain, it also calls the notifier_call with NETDEV_REGISTER
for each existing netdevice and with NETDEV_UP for each netdevice that
is up.  Our can_notifier() will therefore be called not only for newly
created netdevices but also for devices that exist before loading the
can.ko module.
