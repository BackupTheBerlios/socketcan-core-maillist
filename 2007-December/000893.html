Robert Schwebel wrote:
> On Mon, Dec 03, 2007 at 11:29:06AM +0100, Michael Stevens wrote:
>> 'bitrate' definately yes. 'baud' is the wrong unit, see
>> http://en.wikipedia.org/wiki/Baud
> 
> Note that the reference is the CAN Specification here.
> 
>> I would suggest to 'bitperiod' as the most appropriate name. 'time'
>> generally means a point on the time line, while 'period' is the amount
>> of time between two points.
> 
> No, the can spec has quite explicit wording here and we should follow
> it:
> 
> - bit rate
> - bit time
> - sync seg
> - prop seg
> - phase seg1
> - phase seg2
> - sample point
> - information processing time
> - time quantum
> - minimum time quantum
> 
> See CAN Spec 2.0, Part A, Section 8. Whenever these names seem to be
> appropriate, we should use them.

I should be a bit more specific on what might be changed:

"baudrate -> bitrate" breaks the API as it renames:

  SIOC[SG]CANBAUDRATE
  can_baudrate_t

  It is the correct name, technical wise and therefore we should change
  it now.

"bit_time" -> "bittime" is just an internal (cosmetic) change.

  We have already defined bittime (without "_") for custom bittime
  settings and I think the name is OK. Please check
http://svn.berlios.de/svnroot/repos/socketcan/trunk/kernel/2.6/include/linux/can/ioctl.h.

Wolfgang.
