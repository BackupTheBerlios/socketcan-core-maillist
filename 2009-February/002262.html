when it's currently not in use.

debugging output makes sense. I assume that debugging output is 
interesting for development which should(!) be done, when the driver 
gets mainlined.

to see CAN problems that may cause CAN controller resets, controller 
re-initializations and these things. But then the current help message 
would need some update ;-)
