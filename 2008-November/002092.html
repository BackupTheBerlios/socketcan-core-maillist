> Sven Geggus wrote:
>> what are your plans to include real device drivers into mainline? I'm
>> looking forward to see a Kernel, where I could just enable can support
>> and
>> be done. Or even better where distros can be shipped with a socketcan
>> enabled Kernel.
>>   
> 
> With the current CAN netdevice library with sysfs support we made a big
> step for the driver API consolidation.
> 
> There is a TODO list in the trunk
> 
> http://svn.berlios.de/svnroot/repos/socketcan/trunk/TODO
> 
> which needs to be cleaned up as *many* things have already been solved
> and defined with the current implementation.
> 
> I wonder if we should go with the current stuff or what _really_needs_
> to be defined before the CAN netdev library (and associated drivers) can
> touch the mainline?!?
> 
> If missing functionalities can be added without breaking the currently
> defined (sysfs) interface, we should post the drivers and continue the
> development in the mainline Kernel.
> 
> Wolfgang, Marc, all: What's your opinion about this suggested approach?
> Do you think, we are 'ready enough' for mainline?

Yes, with a small set of selected drivers. Some drivers like the PEAK
PCI are not yet read for mainline inclusion and I also prefer to
postpone the inclusion of complex drivers, like for PCMCIA to avoid
unnecessary delays in the review process.

In principle we could do an effort to post the patches before the next
merge window opens already for 2.6.29, but I personally am still short
of time for this project :-(.

Wolfgang.
