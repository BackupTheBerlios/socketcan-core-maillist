No an ioctl() for the CAN network interface as already suggested by 
Wolfgang:

https://lists.berlios.de/pipermail/socketcan-core/2007-October/000695.html

The SJA1000 seems also to support this feature. Controller specific 
setting could be done with the IOCTL request SIOCSCANCTRLMODE. So far we 
have defined:

   cat include/linux/can/ioctl.h
   ...
   #define CAN_CTRLMODE_LOOPBACK   0x1
   #define CAN_CTRLMODE_LISTENONLY 0x2

And it makes sense to add:

   #define CAN_CTRLMODE_SINGLESHOT 0x4
