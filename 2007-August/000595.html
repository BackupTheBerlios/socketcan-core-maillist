That's what I know, too. I simply wanted to make sure that I don't
duplicate someone else's work.

> Looking through the Atmel documentation the controller seems to have
> funny features like "Chaining Mailboxes" in the RX path. Maybe it makes
> sense to create several chains (for SFF, EFF, RTR, etc.) ...

Dunno if that fits into SocketCAN's concepts. At the moment, I tend to 
simply chain mailboxes, e.g. 8 for RX, 8 for TX to create a RX/TX FIFO
buffer. But I don't think you'll see much effect from that, as you
usually don't have that much traffic in a CAN system. In some applications
it might actually increase performance as hardware can receive a new
message even if the previous message hasn't been processed yet.
