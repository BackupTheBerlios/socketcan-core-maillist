The current mkpatch script get's a kernel version (e.g. 2.6.23) as 
commandline parameter, gets the base patch (for the kernel include 
files) from  patches/<kernelversion>.diff and adds the filenames given 
on stdin as patches to this base patch. And all this is printed to stdout.

Example:

mkpatch 2.6.23 < FILES-2.6-ALL > this_is_my2623.patch

The script from Robert generates a new directory with separate patches 
for the selected source files (which are written in the script itself). 
The commandline option is the patch to the Linux kernel (what's that for?).

Both scripts use the strip-src script to remove the KERNEL_VERSION 
depended code.
