Hi Sven,

indeed the current mailine patches only include the CAN core (PF_CAN), 
the two CAN protocols and the VCAN driver. The VCAN driver is a CAN 
netdriver (drivers/net/can) while anything else is a protocol family 
(net/can). Including the VCAN could be considered to be a second step 
for mainlining - but having VCAN as a part of the patch series brings us 
the infrastructure for comming CAN netdrivers also.

So this frist step already contains the second step :-)

And be sure there had been very detailed discussions on the code so far 
- including the drivers for 'real' hardware would extend the discussions 
and would also slow down the mainlining of the PF_CAN.

Posting CAN drivers as an addition for an existing infrastructure would 
be much easier than putting all this into a huge patch series that 
nobody likes to review.
