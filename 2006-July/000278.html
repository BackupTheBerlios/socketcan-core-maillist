Hi Wolfgang,

Wolfgang Grandegger wrote:
> Hello,
> 
> what is the default behavior of a socket if no filter has been defined 
> with setsockopt? Should it receive all messages by default or none?

It receives all messages. If you make any setsockopt()-syscall to set
filters BEFORE the bind()-syscall, you will only get the wanted messages. If
you just say bind() you will get everything.
