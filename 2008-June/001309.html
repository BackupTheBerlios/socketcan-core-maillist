Sure? I always try to keep the SVN source at the head of the kernel 
development.

So you should be able to create a complete patch for all kernel versions 
since ~2.6.13 up to 2.6.26 ... e.g. with

cd socketcan/trunk
./mkpatch 2.6.24 < FILES-2.6-ALL

For 2.6.25 and 2.6.26 you need to exclude the already mainlined patches 
- of course. But using 'strip-src' file-by-file remains working to get 
head of kernel C-files.
