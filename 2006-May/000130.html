Andrey Volkov <avolkov at varma-el.com> writes:

> When I try to merge and build sockcan inside kernel tree,
> I bumped with next problem:
> 
>  include/linux/can/xxxxx.h: version.h: No such file or directory

Surprises me.  The source dir where the .c files are located should be
searched for include files and version.h therefore be found.

> Is including of version.h in headers critical for you?

It was.  When we began developing I wanted to have revision info of
all source files, i.e. C sources and header files in the object files.
The normal static char rcsid[] = "..." doesn't work well for header
files so I have played some tricks with inline assembly.

Since I don't consider this essential anymore I have removed the
include directives and use of the RCSID macro from all header files as
you requested.
