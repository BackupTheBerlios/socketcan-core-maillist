Hi,

I recently tested ip's new features for virtual network interface
creation, because it is now the "modern" way for virtual CAN interfaces
in the SocketCAN project and ontop of linux-2.6.23-rc. While trying to
build iproute2 from the sources, it turned out that the build system is
in a horrible state:

- it assumes the kernel in fixed locations, although it is easily
  possible to find out where the kernel source is these days

- the IPLINK_IOCTL_COMPAT changes the behaviour of the commands, but
  does not change the --help output

- cross compiling is a nightmare

- the packet is not SYSROOT/DESTDIR aware, which makes it a pain to
  package that stuff.

Now before spending major work into fixing these issues properly: would
you be willing to accept a patch series against the latest released
version which properly autotoolizes the packet and fixes the problems
mentioned above?
