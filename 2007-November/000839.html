ACK. The can-utils from the SocketCAN SVN (like candump and canplayer) 
allow to create and replay very nice (minimal - but human readable) 
logfiles. These logfiles can be zipped very small (a student created a 
binary file format for fun which turned out to be bigger when zipped). I 
also wrote some conversion tools for the ASC-CAN-logfiles that are used 
in some Windows CAN Tools.

candump and canplayer can print and receive to/from STDOUT/STDIN so that 
you may put a netcat in between and replay CAN frames on a different 
machine with nearly correct interframe gaps.

Maybe you just can play with the tools to indentify any missing things 
(like cansequence ;-)
