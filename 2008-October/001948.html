Thanks for the patch, but I managed to plug the berlios svn into the
linux-can git repo at pengutronix. I cherry-picked these fixes.

In the meanwhile there are a lot of branches in the repo:

- linux-can.mkl/dev
HEAD of development

- linux-can.mkl/dev.lastmerge
last revision "flattened" into upstream

- linux-can.mkl/git-svn
svn import from berlios svn with "git-svn"

- linux-can.mkl/git-svn.lastmerge
last revision which I have cherry-picked fixes from

- linux-can.mkl/master
linus' tree

- linux-can.mkl/mkl
my personal playground

- linux-can.mkl/upstream
flattened patches, which can be send upstream


The workflow currently is:

- from berlios to git:
  * import via git-svn from berlios into "git-svn" branch
  * cherry-pick fixes by hand into "dev"
  * remove "ifdef KERNELVERSION" stuff

- my developement:
  * work on mkl branch
  * rebase to latest "dev"
  * pull "mkl" into "dev" branch

- generate upstreamable patches:
  * rebase the patches from "dev.lastmerge".."dev" onto upstream
  * use rebase interantive to sort them into the existing patches
   (if needed)

Nevertheless it is possible to follow the berlios-svn development with
git, I still hope that we quickly close the svn or at least use a
git-tree as our primary rcs.

In case someone wants to contribute to the git directly, please send
pull-requests (preferred) or patches against "dev". The at91 stuff is
still under development thus in the "mkl" branch.

cheers, Marc
