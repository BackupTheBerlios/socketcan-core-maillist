That's why I think we should check what's there and integrate it
properly. I'm not religious about the tools in our packet: they have
been there because we needed proper tools for real-world projects.
Customers who have used them in the past need to migrate anyway, once
the drivers are in mainline. If the BerliOS tools have more
functionality in one or the other aspect, let's check what's there and
make something sane out of it (sane as in proper build system,
short/long opts etc). We can for example start with a new major release
number to make proper version dependencies possible.
