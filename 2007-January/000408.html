Wolfgang Grandegger wrote:
>>>   o The bit timing calculation in dev.c does not return the usual BTR0/1
>>>     values for the SJA1000, e.g. BTR0 = 0x03, BTR1 = 0x1C for 125 KB/s.
BTW, Wolfgang, BTR0 = 0x03, BTR1 = 0x1C is an example of such
inappropriate values, since you are kill CAN fault tolerance by setting
up SJW to 1 and phase_seg2 to 2. This values will be ok for a short line
(couple of tens meters), but you could have problems on long lines
(hundreds meters) due to edge pulling or spikes. And you could receive
long line not only by length, but with number of devices too. Some tens
of devices in a net is a usual case.
