I disagree. Let us please define one common, controller-independent
model and make *that one* configurable (supported at all, i.e.
compile-time switch + enabled by the application, i.e. socket-wise
runtime switch). This additional knob will only raise the level of
confusion.

Again, if you only think of debugging, attach to the tap interface
(RT-CAN could export such thing as well like RTnet does). The loopback
as I understand it should emulate the real bus as good as possible for
the sake of distributed CAN applications happen to run on the same node.
