Hi Marc,

i can also see this point.

But if possible i would like to reduce the confusion, when people are 
looking into include/linux/can/* ...

I would see three approaches to solve this:

1. have a single include/linux/can/platform.h for all drivers

2. have a separate directory include/linux/can/platform/* that contains 
e.g. mcp251x.h

3. have a prefix, e.g. pltf_ for e.g. include/linux/can/pltf_mcp251.h

For 3. we could also think about pl_  , platform_  or something else. 
But not the swapped mcp251x_platform.h which won't sort the directory 
entries appropriate at 'ls' ...

Any other comments on this RFC?
