On Thu, Jan 29, 2009 at 07:05:37PM +0100, Oliver Hartkopp wrote:
> i had a short discussion with Urs on this topic and it looks a bit like
> overkill as the SocketCAN tools only run on Linux (and not on BSD,
> Solaris, Irix, True64, whatever) which is AFAIK the main focus for
> autotools. And cross-compiling also works well with 'make CC=...'

Autotoolized packets offer a uniform config interface for distributors.
Wearing my ptxdist maintainer's hat I can state that people who assume
to be smarter than the autotools often cause distributors a lot of pain.

Example: the can-utils makefile is hardcoded to install to
/usr/local/bin (which is obviously wrong). Automake has a sophisticated
policy how to control application installation, including support for
SYSROOT/DESTDIR. There are countless other things, like proper
versioning, dependency management, automatic detection of can.h being
available in the toolchain or not etc. You get the picture. Check
canutils' configure.ac for details.

> The generated stuff from autotools always look magic to me.

Reading 'info automake' is a good start if you want to learn how it
works.
