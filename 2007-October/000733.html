Chris, I do not have "ISO11783-5:2001", but I have "J1939-81 Network
Management", which is seems to be the same. I have not found any
explicit requirement there for using single shot mode.
I have also talked to our J1939 developer and he confirmed to me that
our address claim implementation do not requires single shot mode. 
Can you please verify that issue?


My understanding for aim of single shot mode is to avoid Tx pending if
no peer is connected or arbitration lost and avoid error active/bus off
if something is wrong with medium or configuration of CAN controller.
Using such a "safe" approach to communicate with CAN bus we make the
least disturbance to the CAN bus and ourselves in case something is
wrong.
Are there any other aspects I have forgotten?

For me single shot is rather CAN controller feature, not per message
feature even if some CAN controller allow to activate this mode for the
single message.
