Cliff Brake wrote:
> I was wondering if you would consider converting to autotools for
> building the user space apps in socketcan?  This this makes cross
> compiling and packaging much easier for a number of environments such
> as OpenEmbedded.
>   

Hello Cliff,

i had a short discussion with Urs on this topic and it looks a bit like 
overkill as the SocketCAN tools only run on Linux (and not on BSD, 
Solaris, Irix, True64, whatever) which is AFAIK the main focus for 
autotools. And cross-compiling also works well with 'make CC=...'

The generated stuff from autotools always look magic to me.

If it helps you, i would like to put your added files on the SVN without 
removing the current Makefiles (that are overwritten by autotools 
anyway). So people can choose, if they would like the current behaviour 
or if they would like to use autotools when they get a clean SVN checkout.

I know that this would mean to modify two files when adding new tools - 
but this does not look problematic to me as there is not much changing 
traffic.

Would this be ok for you?
