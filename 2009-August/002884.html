When we follow the un*x tool directive 'somehow similar' functionality is a no
go. If you would have 'ls' and 'dir' with completely different commandline
parameters the user get's lost. This is not the successful un*x tool approach.

E.g. checking two of the four tools in the ptx canutils

canconfig: creates an additional interface to configure the deprecated sysfs
configuration interface - is this still needed?

candump: has a name clash with the BerliOS candump which has much more
functionality and fit's to the rest of the suite like canplayer, etc.

How do we proceed with these items when collecting the 'floating stuff'
without confusing the user?
