> files) from  patches/<kernelversion>.diff and adds the filenames given
> on stdin as patches to this base patch. And all this is printed to stdout.

Yes, and it should work for old and new kernels (with the CAN core
included) if all files are properly included and it can then replace
mkpatch at all. Note that these patches can not be used for kernel
inclusion because of the missing dependencies.

> Both scripts use the strip-src script to remove the KERNEL_VERSION

Yes. For the kernel inclusion we want do work on the SVN trunk and after
some modification we need to re-generate a usable patch series to be
checked in. This requires some tricks to reproduce a proper patch stack
and should be restricted to the new files we really want to be included.
I would provide a separate patch for that purpose.
