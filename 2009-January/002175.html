if you look at the mcp251x code you can see that you should at least
see a message "probe failed" if some reason the driver cannot
initialize itself (or a message "probed" if it's all right). So I
guess that, for some reason, the SPI subsystem doesn't run the probing
function. Just to be sure you see all the messages on the console do
something like "echo 100 > /proc/sys/kernel/printk"
