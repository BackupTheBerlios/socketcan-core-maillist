Since 2.6.24 the virtual CAN driver 'vcan.ko' uses the netlink-API to 
create vcan-software devices at runtime.
As the netlink-API has been introduced in 2.6.24,  it is no real 
surprise, that vcan.c does not compile on earlier kernels.

For Kernels 2.6.23 and below the number of needed vcan devices can be 
specified on the module commandline at loading time of vcan.ko (default 
= 4).

But maybe the compilation should produce some reasonable error output 
that tells the user that it just doesn't work with older kernels, so he 
does not get confused about it ...
