On Mon, Jun 09, 2008 at 11:08:53AM +0200, Wolfgang Grandegger wrote:
> We do want to provide "good" default settings by simply setting the
> bitrate.

Can be done with our model; simply provide the CIA recommended values as
a default which can be overwritten.

> > What's the advantage of SJA-specific btregs vs. standard provided
> > parameters?
>
> The btregs are not SJA-specific. The 4 bytes can be used to setup the
> relevant hardware register. As I see it: handling standard provided
> parameters is too sophisticated for the kernel and may still fail.

Not all CAN controllers have btregs, and it is an arbitrary hardware
dependend byte interface to userspace. Remember, kernel drivers are
about abstraction...

> These values are already tweaked for a special hardware and not good for
> other CAN controllers like the SJA1000.

Why do you think SJA can *not* implement the equivalent? The model comes
from the CAN 2.0A Standard, so *any* standard compliant controller
should be able to fulfill it.

> You do a hardware specific setting in a hardware independent way. I
> know, it's needed for the MSCAN, at least but do we need all this in
> the kernel just because of a very few wired CAN controllers out there.

Please prove me wrong, but exposing an arbitrary byte interface to
userspace is much, much more weird than using the official standard
model.
