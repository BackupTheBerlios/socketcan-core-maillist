It's not pretty printing. A user-space application, like "ip" does, can query the 
CAN bit-timing constants using the netlink message type

	[IFLA_CAN_BITTIMING_CONST]
				= { .len = sizeof(struct can_bittiming_const) },

and it would be nice if it would contain the clock frequency as well. But the
old criticism is still valid, I thought it was something less harmful. But
"clock" does also not belong to "struct can_bittiming", which is also used to
set the bit-timing. Looking for a more clever solution. At least the current
implementation for SJA1000 is not correct.
