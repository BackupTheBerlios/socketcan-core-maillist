> I assume alignment is ok for definitions, but not within functions?
> I consulted the Documentation/Coding-style, but I did not find
> the exact answer.

The rationale behind not aligning code is: If you add later something
longer to the code you have two choices: either realign everything or
don't align the new longer variable. Realigning is evil, because it
breaks "git blame". So it's better not to align in the first place.
