> Hello,

> the attached patch adds the CAN utility canconfig.c permitting to
> set/get CAN controller options via SIOC[SG]CAN* ioctl requests. This
> is for example necessary, to set the baudrate for the MSCAN controller.

Hi Wolfgang,

thanks for your work!
I don't have general objections to your tool as it was missing for a
long time.
But i'm not sure, if we should consolidate the current typedef-hell in
can/ioctl.h ?!?

This would definitely be a job-stopper for kernel mainlining, as new
introduced typedefs are declared to be evil.

Of course this is something that can even be solved *after* having
canconfig for a time but i'm still unsure if we should try to find a
better solution for the typedefs (in mscan) *before* we're providing the
tools which would cause the other drivers to migrate.

That's my only concern ...
