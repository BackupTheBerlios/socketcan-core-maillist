Hi, 

As the socket-can code is becoming more and more complete, we should
care a little bit more about the upstream requirements. I just had a
short look into the code base and found these issues: 

- You still don't use kernel coding style, which is an absolute must. No
  four spaces for indents, just one tab.

- These compatiblity ifdefs for older kernels should go away, maybe not
  now but definitely before the code is being posted somewhere. If you
  have to support a kernel series, make a separate patch series for that
  one. 

- Don't invent your own DBG(), just use pr_debug(). 

- Don't invent your own lists, the kernel has a pretty good 
  default implementation for them. 

- timeval2jiffies should go somewhere into lib/.
