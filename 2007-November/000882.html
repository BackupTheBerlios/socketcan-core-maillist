Yes, and there's no need to set this manually. A simple script could
wrap this and make it more human readable. No need for a complicated
parser in kernel space. I find your first idea acceptable.

> 
> >>>>   # ls can_stats/

> >>> sync with what's already there.
> >> You prefer "can_statistics"? Fine for me.
> > 
> > No, only "statistics", to make it identical to ethX.
> 
> There is no simple and clean way to add files to this directory,
> because it's an attribute group defined in "net/core/net-sysfs.c".
> And I'm also afraid of name clashes. I personally prefer names with
> the prefix can_, e.g. can_state, can_mode, can_statistics.

That's what I think, too. CAN specific attributes that no other
networking device has should be prefixed "can_". And the directory
"can_statistics" should only contain those attributes that are not
already present in /sys/class/net/canX/statistics/.
