I suppose we should discuss two topics: 

Coding Style 
------------

If we want mainline integration, the question is when to cleanup the
current code base. It still is terribly messed up with regard to kernel
coding style, which is pretty bad because it will definitely break the
svn history when we do something against it. Nevertheless, before any
mainline action can be done it has to be cleaned. And, of course, there
are quite some code parts which have to be rewritten using modern kernel
infrastructure. 

Deployment / Patch Generation
-----------------------------

I've started some scripts to integrate the tree into some random kernel,
but the question is which way we want to go with the result. The only
sane solution for upstream posting is to make topic separated patch
stacks, so what we probably need if we want to stay with the current
solution to have files and not patches in svn is scripts which
automatically generate patch series from the files. Any comments?
