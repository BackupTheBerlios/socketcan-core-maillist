Hello,

some thought on how to get the CAN device interface and a few drivers
into kernel 2.6.30:

- For patch preparation I tend to reactivate my /trunk/mkseries-dev,
  extracting code from the SVN repository and preparing and committing
  a series of patches. It should work fine for this purpose. An
  alternative would be to use GIT immediately.

- For the initial inclusion process and review we should only provide a
  few simple drivers, e.g.:

  SJA1000: ems_pci.c, kvaser_pci.c and PXA270.
  MSCAN for MPC5200.

  My criteria here is simplicity and test ability. It should not make
  the review more delicate and we need people for testing the code and
  drivers. Who is available?

  Other drivers could be committed when the initial CAN device interface
  has been accepted for inclusion.

- Further code review and cleanup like checking coding style and
  copyright notes.

- And we need documentation.

What do you think.

Wolfgang.
